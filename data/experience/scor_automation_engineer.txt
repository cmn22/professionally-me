1. Databricks Non-Regression Testing Framework:

Context:
SCOR relies heavily on data pipelines built on Databricks for actuarial modeling and reporting. Any unintended change in data outputs could have significant financial and business implications. Manual validation was slow and error-prone.

What I Built:
Designed and implemented a non-regression testing framework to automatically compare "expected" vs "actual" datasets generated by data pipelines.

Key Features:
- Automated dataset comparison using PySpark
- Column-level and row-level validation
- Threshold-based validation for numerical fields (to allow acceptable variance)
- Exact matching for categorical and string fields
- Detection of:
  - Missing rows
  - Additional rows
  - Schema mismatches
  - Value-level discrepancies

Technical Implementation:
- Used PySpark DataFrames for distributed comparison of large datasets
- Implemented comparison logic for:
  - Numeric columns → tolerance-based comparison
  - String columns → strict equality
- Built reusable comparison functions to standardize validation across pipelines
- Generated structured output reports highlighting:
  - Differences
  - Percentage deviation
  - Columns impacted

Example Logic:
- If difference in numeric column > threshold → flag as warning or error
- If categorical mismatch → mark as regression
- If schema mismatch → fail test immediately

Outcome:
- Reduced reliance on manual validation
- Enabled early detection of regressions in data pipelines
- Improved confidence in production data outputs

2. Threshold-Based Alerting System:

Problem:
Not all numerical differences indicate a failure. Small variations are acceptable depending on the context.

Solution:
Implemented a threshold-based alerting system to classify discrepancies into:
- Acceptable (within tolerance)
- Warning (slightly above threshold)
- Critical (significant deviation)

Details:
- Configurable thresholds per column
- Percentage-based and absolute difference checks
- Flexible rules depending on data type and business requirements

Impact:
- Reduced false positives
- Made testing system more practical for real-world data variability

---

3. Dockerization of Testing Framework:
