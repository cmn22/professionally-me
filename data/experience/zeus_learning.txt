Title: Software Engineer
Company: Zeus Learning
Duration: June 2022 - August 2023
Location: Mumbai, India

Overview:
Worked as a Software Engineer with a strong focus on full-stack development, contributing to the design and development of an educational examination platform. The platform enabled teachers to create diverse question types and allowed students to attempt assessments with automated evaluation.

---

Application Context:

The system was an examination platform supporting multiple dynamic question formats within a single assessment. A question paper could contain a mix of different question types, allowing flexible evaluation of students.

Supported Question Types:
- Fill in the blanks
- One-word answers
- Paragraph-based answers
- Image labeling questions
- Match-the-following (criss-cross format)

Key Features of the Platform:
- Teachers could create and configure question papers with multiple question types
- Students could attempt exams through an interactive interface
- Automated evaluation and scoring mechanisms
- Flexible composition of question papers with heterogeneous question formats

---

Core Responsibilities:

- Developed reusable frontend and backend components to support multiple question types
- Contributed to both backend (Node.js) and frontend (React) development
- Worked on database interactions and data migration strategies
- Ensured accessibility compliance in the frontend
- Led a small team and owned end-to-end delivery of specific modules

2. Database Interaction and ORM Usage:

Context:
The application relied on a PostgreSQL database to store questions, answers, and evaluation data.

What I Did:

- Implemented efficient data retrieval and storage mechanisms
- Structured data models to support multiple question formats

Responsibilities:

- Ensuring data consistency across different question types
- Optimizing data access patterns where needed

3. Migration to Amazon DynamoDB:

Context:
As part of system evolution, there was a need to transition certain data workflows to a NoSQL database (Amazon DynamoDB) for scalability and performance.

What I Did:
- Wrote scripts to migrate data from PostgreSQL to DynamoDB
- Handled transformation of relational data into NoSQL-compatible formats
- Ensured data integrity during migration

Challenges:
- Mapping relational schemas to key-value/document-based structures
- Maintaining consistency during migration
- Handling edge cases in data transformation

Outcome:
- Enabled the system to leverage DynamoDB for scalable data storage
- Improved performance for specific use cases
